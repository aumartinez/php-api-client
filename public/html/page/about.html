About

This demo consumes the Super Hero API, to read or learn more about this API, please feel free to visit their official website at: https://superheroapi.com/index.html.

The demo is supposed to support the below:

When performing a search, in case there is an error it should display the error received from the API, if there aren’t any results it should display a custom message, otherwise it should display all the returned results from the API, displaying at least the image, name, and gender of the character as a minimum.

Detailed information from the found superhero or villain of choice. In case it doesn’t have an image associated, show one by default.

The search component should still be available and if used should behave like the initial search page going back and displaying the results. The search component should keep the last used search criteria.

The URL of the detail page should be /details/XXX where XXX is the id of that specific superhero or villain character and when performing the search the URL should always be the root.

When using the detail pages, sometimes can happen that the URL could be misspelled or an invalid one when manually navigating to this page through the browser bar. When nothing is found or tried to reach a not build resource, a nice Not Found page should be displayed
